# Workflow — what order do I run everything in?

The Readme has a one-shot Quickstart for the impatient. This doc is the full step-by-step including the branch points (all-projects vs single-project) and the "exit the toolkit" step.

Run everything from inside `~/insights-badger/`.

## One-time setup

1. **`./scripts/list-projects.sh`** — see what's live in `~/.claude/projects/` vs already parked in `~/backup/projects/`. Safe to run anytime; doesn't move anything.

2. **`./scripts/backup-projects.sh`** — parks every project under `~/backup/projects/` and clears `~/.claude/usage-data/` caches. `/insights` now has nothing to analyze. This is the "prepare a clean slate" step.

## Path A — all projects (automated loop, recommended)

3. **`./scripts/insights-per-project.sh`** — for every parked project, in a loop:
   - restore it to `~/.claude/projects/`
   - clear `~/.claude/usage-data/`
   - drive `/insights` via the expect TUI script with full output streaming to your terminal
   - copy `report.html` to `~/insights-badger/results/<encoded-name>.html` (e.g. `-Users-michaeldimmitt-scripts.html`)
   - re-park the project

   Set `STOP_ON_FAIL=1 ./scripts/insights-per-project.sh` to halt on the first failure and leave that project live so you can re-run the expect driver manually for debugging.

## Path B — one project at a time (manual)

3. **`./scripts/restore-one.sh <project-substring>`** — restores one parked project, clears `usage-data/`, and runs `/insights` against it via the expect TUI driver. The report is saved to `~/insights-badger/results/` and the project is **left LIVE** so you can poke around (run `/insights` again, open new sessions, etc.). Re-park with `./scripts/backup-projects.sh` when done.

   Use this when you want to iterate on a single project's report, or when the loop hit a failure and you want to retry just one.

## When you're done

4. **`./scripts/restore-all.sh`** — moves every parked project back into `~/.claude/projects/` and clears `usage-data/` so the normal multi-project `/insights` works again.

   `insights-per-project.sh` re-parks each project after running it, so after the loop **everything is still hidden** from `/insights`. This is the command that exits the toolkit and gives you your normal Claude Code state back.

## Anytime

- **`./scripts/list-projects.sh`** — scans both `~/.claude/projects/` and `~/backup/projects/` and tells you where everything is. Useful for sanity checks mid-flow.

## Viewing the reports

**`./scripts/open.sh results/*`** — opens every generated HTML report in your browser. Cross-platform (macOS `open`, Linux `xdg-open` or `wslview` on WSL, Windows Git Bash `start`).

You can also pass specific files or globs:

```bash
./scripts/open.sh results/-Users-michaeldimmitt-scripts.html
./scripts/open.sh 'results/*scripts*.html'    # quoted glob, expanded by the script
```

## Directory layout

```
~/.claude/projects/-Users-michaeldimmitt-foo/   <- LIVE: source transcripts
  <session-id>.jsonl
~/.claude/usage-data/                           <- DERIVED: regenerated by /insights
  session-meta/  facets/  report.html
~/backup/projects/-Users-michaeldimmitt-foo/    <- PARKED: hidden from /insights
~/insights-badger/results/<encoded-name>.html   <- per-project /insights output
```

The encoded directory name is the project path with `/` replaced by `-` (leading `-` included). Example: `/Users/michaeldimmitt/scripts` → `-Users-michaeldimmitt-scripts`.
